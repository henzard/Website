# ğŸ’œ Using with Lovable

[Lovable](https://lovable.dev) (formerly GPT Engineer) is an AI website builder. Here's how to use these rules:

## Step 1: Copy the Rules Summary

Copy this text and paste it at the START of your Lovable chat:

```
IMPORTANT RULES FOR THIS PROJECT:

1. PROJECT SETUP
- Always ask for requirements before coding
- Create a color palette first
- Use React + TypeScript

2. SECURITY
- Never store sensitive data in plain text
- Validate all user inputs
- Use proper authentication

3. CODE QUALITY
- Use clean architecture (separate UI, logic, data)
- Handle all errors gracefully
- Show loading states

4. USER EXPERIENCE
- Mobile-first responsive design
- Minimum 44x44px touch targets
- Show clear error messages

5. PERFORMANCE
- Lazy load images
- Minimize bundle size
- Cache where possible

6. ACCESSIBILITY
- All images need alt text
- Forms need labels
- Keyboard navigation must work

7. FORMS
- Validate on blur, then on change
- Show inline error messages
- Disable submit while processing

8. DATABASE (if using)
- Use camelCase for field names
- Never dump JSON into fields
- Index frequently queried fields

Please follow these rules for everything you build.
```

# ğŸ’» Using with Replit

[Replit](https://replit.com) has an AI assistant. Here's how to use these rules:

## Step 1: Create a New Replit

1. Go to [replit.com](https://replit.com)
2. Click `Create Repl`
3. Choose `React` or `Next.js` template
4. Name your project

---

## Step 2: Add a Rules File

1. Create a new file called `AI_RULES.md`
2. Paste this content:

```markdown
# AI Development Rules

## Before Writing Code
- Ask clarifying questions about requirements
- Plan the architecture first
- Consider mobile users

## Code Standards
- TypeScript strict mode
- Functional components only
- Custom hooks for shared logic

## Security Requirements
- Validate all inputs
- Sanitize data before display
- Use environment variables for secrets

## UI/UX Requirements
- Mobile-first design
- Loading states for all async operations
- Error messages must be user-friendly
- Touch targets minimum 44x44px

## Performance Requirements
- Images must be optimized
- Lazy load below-fold content
- Bundle size under 200KB initial

## Form Requirements
- Client + server validation
- Inline error messages
- Disabled state while submitting

## Accessibility Requirements
- All images need alt text
- Forms need proper labels
- Color contrast 4.5:1 minimum
- Keyboard navigation support

IMPORTANT: Reference this file before implementing any feature!
```
